发表于 2017-10-30

make 和 Makefile

可以查看这边博客了解其大致用法。http://blog.csdn.net/xkq_lyx/article/details/17800791

Makefile 文件

/ Makefile /

DO NOT EDIT THIS FILE

include build/core/main.mk

DO NOT EDIT THIS FILE

内容指向 build/core/main.mk

build/core/main.mk

1 文件大，引用多，如何阅读呢？牢记文件结构，以此分析。

2.1 编译系统往往不只有一颗依赖树，比如 “make” 和 “make sdk” 。
2.2 原则上 Make 程序会对 makefile 中的内容按照顺序进行逐条解析。
典型的解析流程为：

* 变量赋值、环境监测等初始化操作
* 按照规则生成所有依赖树
* 根据用户选择的依赖树，从叶高根逐步生成目标文件

3 阅读该文件获得默认根节点 droid

This is the default target. It must be the first declared target.

.PHONY: droid
DEFAULT_GOAL := droid
$(DEFAULT_GOAL): droid_targets

不过这里的 droid 还是一个空内容，相当于预先占用了一个位置来保证他第一个出现。

mac_sdk_versions_supported := 10.8 10.9 10.10 10.11

根据版本号下载响应版本放入 ~/lib 中，并设置软连接
